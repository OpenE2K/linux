1. ИМЯ И ЗАПУСК ДРАЙВЕРА.
	Драйвер включен в ядро ОС (drivers/net/rdma_net, CONFIG_RDMA_NET).

2. ОПИСАНИЕ ДРАЙВЕРА.

	Сетевой драйвер контроллера УПДП обеспечивает независимую передачу и
	прием данных по паре независимых каналов. Для этого он создает два
	сетевых интерфеса - sn0, sn1.
	Работа с ними осуществляется с посредством стандартных функций работы с
	сетевыми устройствами.
 

3. СХЕМА ОРГАНИЗАЦИИ БЕЗДИСКОВЫХ КЛИЕНТОВ ВК МВС/С НА ОСНОВЕ КОНТРОЛЕРА УПДП.
	ВК состоит из сервера (узел 0 выспупает в качестве nfs, dhcp серверов) и
	1-3 клиентов.
	Для создания ВК на основе контролера УПДП следует выполнить следующую
	последовательность действий:
	- "поднять" сетевой интерфейс sn0 на узле "0" с помощью скрипта
	/sbin/set_network.
	- исполнить скрипт /mcst/diskless-client-utils/setup.


	Скритп создает (удаляет, корректирует) файлы (в части  сетевых
	интерфейсов sn ):

    - сервер (узел 0):
      /etc/rc.d/init.d/dhcp
      /etc/dhcpd.conf
      /etc/hosts
      /etc/sysconfig/network-device/ifconfig.sn0/ipv4
      /etc/sysconfig/network-device/ifconfig.sn1/ipv4
      /etc/sysconfig/network-device/ifup


    - клиент 1 (узел 1):
      /etc/rc.d/init.d/dhcrelay
      /etc/hosts
      /etc/sysconfig/network-device/ifconfig.sn0/ipv4
      /etc/sysconfig/network-device/ifconfig.sn1/ipv4
      /etc/sysconfig/network-device/ifup

    клиент 2 (узел 2):
      /etc/hosts
      /etc/sysconfig/network-device/ifconfig.sn0/ipv4
      /etc/sysconfig/network-device/ifconfig.sn1/ipv4
      /etc/sysconfig/network-device/ifup

    клиентов 3 (узел 3):
      /etc/hosts
      /etc/sysconfig/network-device/ifconfig.sn0/ipv4
      /etc/sysconfig/network-device/ifconfig.sn1/ipv4
      /etc/sysconfig/network-device/ifup

      Сетевые интерфейсы sn1 клиентов 1 (узел 1) и 2 (узел 2), sn0 клиента 3
      (узел 3) "поднимаются" по протоколу bootp.  Т.к. клиент 2 не находися в
      подсети сервера на клиенте 1 запускается ретранслирующий агент dhcrelay,
      предназначен для передачи DHCP запросов из подсетей, где нет собственного 
      сервера DHCP в подсети содержащие такие сервера.
      Сетевые интерфейсы sn0 клиентов 1 (узел 1) и 2 (узел 2), sn1 клиента 3
      (узел 3) "поднимаются" скриптом /etc/sysconfig/network-device/ifup.
      
      Схема сетевой организации вычислительного комплекса на основе сетевых
      интерфейсов sn приведена на рис.1. 


  
             --- 192.168.x+1.y       192.168.x+1.y+1 ---
	    | 0 |-----------------------------------| 3 |
	     ---  sn1                           sn0  ---
192.168.x.y   |sn0                                 sn1| 192.168.x+3.y
	      |                                       |
	      |                                       |
	      |                                       |
	      |                                       |
	      |                                       |
	      |                                       |
	      |                                       |
	      |                                       |
192.168.x.y+1 | sn1                                sn0| 192.168.x+3.y+1
             ---  sn0                           sn1  ---
	    | 1 |-----------------------------------| 2 |
	     --- 192.168.x+2.y       192.168.x+2.y+1 ---
                               Рис. 1

	Каждый узел находится в двух подсетях. Для доступа к "дальным" клиентам 
	в таблицы маршрутизации узлов добавлятся записи:

	- сервер (узел 0):
	route add -host 192.168.x+3.y+1 gw 192.168.x+1.y+1
	route add -host 192.168.x+2.y+1 gw 192.168.x.y+1
	route add -net 192.168.x+2.0 netmask 255.255.255.0 gw 192.168.x.y+1

	- клиент 1 (узел 1):
	route add -host 192.168.x+1.y+1 gw 192.168.x.y
	route add -host 192.168.x+3.y gw 192.168.x+2.y+1

	- клиент 2 (узел 2):
	route add -host 192.168.x.y gw 192.168.x+2.y
        route add -host 192.168.x+1.y gw 192.168.x+3.y

	- клиент 3 (узел 3):
	route add -host 192.168.x+2.y gw 192.168.x+3.y+1
        route add -host 192.168.x.y+1 gw 192.168.x+1.y



	
	Ввиду того, что в данной схеме присутствуют транзитные узлы оптимальное
	распределение сетевых потоков возлагается на разработчика ФПО.
